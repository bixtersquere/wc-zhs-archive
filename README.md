# wc-zhs-archive

## 项目简介

`wc-zhs-archive` 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为一个通用存档空间，集中存放各类静态 HTML 页面内容。

## 目录说明

```
wc-zhs-archive/
├── pages/          # 存放各归档 HTML 页面的子目录
├── resources/      # 页面共用的资源文件（如样式、脚本等）
├── index.html      # 仓库根目录的入口页面（如有）
└── README.md       # 本说明文件
```

- **pages/**：每个归档页面以独立子目录形式存放，便于管理和引用。
- **resources/**：存放多个页面可能共用的 CSS、JavaScript、图片等资源。
- **index.html**：可选的根入口页面，用于导航或索引。

## 页面归档说明

本仓库中所有 HTML 页面均为独立静态文件，不依赖外部服务器或特定后端服务。页面内容可能包括但不限于：

- 信息整理与展示
- 工具或功能页面
- 存档性质的静态副本

每个页面在 `pages/` 下拥有独立子目录，内部包含 `index.html` 及相关资源文件。页面之间互不依赖，可单独访问。

## 使用方式

您可以通过以下方式浏览本仓库中的页面：

1. 直接克隆仓库到本地，在浏览器中打开对应 `pages/` 子目录下的 `index.html`。
2. 通过 GitHub Pages 等静态托管服务发布后，访问对应页面链接（需自行配置）。

## 维护说明

- 本仓库为归档性质，更新频率不固定。
- 新增页面时，请在 `pages/` 下创建新目录，并确保页面资源完整。
- 如需修改或删除已有页面，请保持仓库结构清晰，避免遗留无用文件。
- 欢迎提交 Issue 或 Pull Request 讨论归档内容或结构改进。
